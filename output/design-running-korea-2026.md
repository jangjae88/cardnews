# Design Brief: 국내 러닝 트렌드 및 러닝제품 트렌드 2026ver

작성일: 2026-06-14  
담당: design  
입력 자료:
- `output/research-running-korea-2026.md`
- `output/analysis-running-korea-2026.md`
- `output/writing-outline-running-korea-2026.md`
- `DESIGN.md`
- `AGENTS.md`

## 결론

1. 전체 톤은 EARLYSHINE Warm/Tan 기반의 컴팩트한 마케팅 스튜디오형 러닝 리포트 캐러셀로 확정한다. 검정, 크림, 탄을 기본으로 쓰고 오렌지는 러너 동선, 핀, 위험/주의 포인트에만 제한적으로 쓴다.
2. 1~6장은 각각 `러너 동선 맵`, `문화/대회 4분면`, `앱/인증 흐름도`, `오프라인 거점 미니맵`, `Before-During-After 제품 매트릭스`, `기회/주의 체크리스트`로 구성한다. 7장은 EARLYSHINE 중앙 로고 엔딩이다.
3. 모든 카드 우상단에는 텍스트 대체가 아닌 실제 로고 이미지 `assets/logo/main_a_transparent.png`를 넣는다. 카드 7은 우상단 미니 로고를 유지하면서 중앙에 같은 로고를 크게 배치한다.

## 디자인 시스템

### 한 줄 톤앤매너

다크 러닝 코스 위에 탄 라인워크와 크림 타이포가 얹힌, 차갑지 않은 2026 러닝 비즈니스 리포트.

### 캔버스

- 크기: 1080 x 1350px
- 카드 수: 총 7장
- 본문 카드: 1~6장
- 엔딩 카드: 7장
- 안전 여백: 상/좌/우 58~64px, 하단 64~76px
- 주요 콘텐츠 최대 폭: 960px
- 표/지도/흐름도는 카드 하단 52~58% 영역 안에 배치
- 큰 제목은 2~4줄, 제목 영역은 상단 28~34% 안에서 마감

### 컬러 팔레트

- Base Black: `#0F0E0C` 전체 배경
- Panel Black: `#16140F` 패널/표/칩 배경
- Ink Cream: `#F3EDDE` 제목/주요 텍스트
- Muted Tan: `#9F927A` 보조 문구/출처/눈금
- Tan: `#D4B896` 핵심 라인/강조 텍스트
- Tan Deep: `#B79A6E` 보더/서브 라인
- Run Orange: `#D96035` 러너 동선, 핀, 작은 배지
- Burnt Orange: `#B85A32` 오렌지 보조선
- Warning Red: `#C36A5C` 주의 체크 전용, 과다 사용 금지

운영 규칙:
- 카드 배경은 대부분 `#0F0E0C`로 통일한다.
- 크림 배경 풀카드는 쓰지 말고, 크림은 텍스트와 작은 패널 하이라이트에 쓴다.
- 오렌지 면적은 카드 전체의 8~12% 이내로 제한한다.
- 보라 그라데이션, 파스텔, 귀여운 러너 캐릭터, 손그림 라인, 노트 템플릿, 장식용 블롭 금지.

### 타이포그래피

- 기본 서체: Pretendard Variable
- 카드 제목: 900 Black, 76~98px, line-height 0.96~1.05, tracking `-0.01em`
- 보조 문장: 600~700, 25~30px, line-height 1.35~1.5
- 섹션 국문: 800 ExtraBold, 26~34px
- 영문 라벨: 600~700, 16~19px, uppercase, tracking `0.26em~0.30em`
- 칩/표 텍스트: 700~800, 19~25px
- 근거/출처/주의: 500~700, 14~17px

운영 규칙:
- 카드당 최종 문구는 제목 1개, 보조 1개, 도표 라벨 중심으로 제한한다.
- 표 안 문구는 8~15자 단위로 끊는다.
- 숫자는 굵게 쓰되 훅으로 과장하지 않는다.

## 반복 컴포넌트

### Header Lockup

- 좌상단: `01 / 07` + 넓은 자간 영문 라벨
- 영문 라벨 예시: `RUNNING KOREA 2026`, `CULTURE`, `PLATFORM`, `PLACE`, `PRODUCT`, `CHECKLIST`, `EARLYSHINE`
- 우상단: 실제 로고 이미지 `assets/logo/main_a_transparent.png`
- 로고 권장 크기: 본문 카드 가로 118~150px, 높이는 원본 비율 유지
- 로고 위치: top 58px, right 58px

### Footer

- 상단에 `Tan Deep` 1px 분할선
- 좌측: 출처/주의 1줄
- 우측: `01 / 07` 형식 페이지 표기
- 카드 7도 총 장수 기준 `07 / 07`

### Diagram Style

- 라인: `#D4B896`, 2~3px
- 보조선: `rgba(212,184,150,0.32)`, 1px 또는 점선
- 핀/활성 노드: `#D96035`
- 패널 radius: 8px 이하
- 그림자 대신 얇은 보더와 명도 차이로 구분

## 사진 사용 규칙

사용 가능한 사진:
- `output/photos/running-korea-2026-night.png`
- `output/photos/running-korea-2026-crew.png`
- `output/photos/running-korea-2026-products.png`

사용 위치:
- 카드 1: `running-korea-2026-night.png`를 풀블리드 배경으로 사용. 상단/좌측 텍스트 영역에 `rgba(15,14,12,0.58~0.72)` 오버레이를 둔다.
- 카드 2: `running-korea-2026-crew.png`를 하단 또는 우측 34~40% 스트립으로 크롭. 인물 얼굴이 잘리지 않게 하고 텍스트는 사진 위에 직접 많이 올리지 않는다.
- 카드 5: `running-korea-2026-products.png`를 매트릭스 배경 질감 또는 우측 상단 크롭으로 사용. 제품 매트릭스 라벨이 사진에 묻히지 않도록 어두운 패널을 먼저 깐다.

사진 톤 보정:
- 채도 -8~-14%, 밝기 -10~-18%, 콘트라스트 +6~+12
- 전체에 웜 블랙 오버레이를 얹어 `DESIGN.md`의 Warm/Tan 라인과 맞춘다.
- 사진은 3장을 넘기지 않는다.

## 카드별 레이아웃

### 카드 1. Hook

역할:
- 러닝을 무료 운동이 아니라 러너가 반복해서 지나는 생활권 동선 비즈니스로 다시 보게 만든다.

레이아웃 타입:
- 풀블리드 야간 러닝 사진 + 대형 훅 + 동선 칩 맵.

구성:
- 배경: `running-korea-2026-night.png` 풀블리드.
- 좌상단 헤더 아래에 큰 제목 영역.
- 중앙 하단에 굵은 러닝 코스 라인 1개를 S자 또는 사선으로 배치.
- 코스 라인 위에 칩 5개: `크루`, `대회`, `앱`, `매장`, `회복제품`.
- 칩은 크림/탄 패널, 활성 핀은 오렌지.

주의:
- 첫 장에는 추정 숫자를 넣지 않는다.
- `1000만`, `1조`, `국민 5명 중 1명` 계열 표현 금지.

### 카드 2. Culture

역할:
- 러닝크루와 대회 접수 경쟁이 러닝을 참여형 경험으로 키우는 흐름을 보여준다.

레이아웃 타입:
- 4분면 리포트 카드 + 러닝크루 사진 스트립.

구성:
- 상단: 짧은 제목과 보조 문장.
- 중앙: 2x2 매트릭스.
- 4칸 라벨: `생활체육`, `러닝크루`, `대회 접수`, `참가 경험`.
- 하단 또는 우측: `running-korea-2026-crew.png` 얇은 크롭.
- 작은 근거 칩: `생활체육 참여율 62.9%`, `전년 대비 +2.2%p`, `KSPO 러닝크루 언급`.

주의:
- 수치는 배경 근거 칩으로만 작게 배치한다.
- 모든 대회가 즉시 마감된다는 식의 일반화 금지.

### 카드 3. Platform

역할:
- 기록 인증과 앱 접수가 러너의 소속감과 대회 경험을 이어주는 구조를 보여준다.

레이아웃 타입:
- 앱/인증 흐름도.

구성:
- 배경: 다크.
- 중앙에 4단 흐름도: `러닝 기록` → `디지털 카드` → `접수/인증` → `커뮤니티`.
- 각 단계는 실제 서비스 화면 복제가 아닌 추상 UI 카드로 표현한다.
- `디지털 카드` 노드는 Tan 패널, `접수/인증` 노드는 오렌지 핀으로 강조.
- 우측 하단에 작은 배지형 근거: `카카오-러너블 러너스 카드`, `Runable 접수`.

주의:
- 플랫폼 효과를 참여 증가로 단정하지 않는다.
- 카카오, 러너블, JTBC 등의 실제 로고는 쓰지 않고 텍스트 근거로만 표기한다.

### 카드 4. Place

역할:
- 러너가 뛰기 전후로 들르는 오프라인 거점이 늘어나는 장면을 설명한다.

레이아웃 타입:
- 도심/한강 러닝 코스 미니맵.

구성:
- 상단: 제목 영역.
- 중앙: 추상 지도. 실제 지도 타일은 쓰지 않고, 강/도로/코스를 탄 라인워크로 표현한다.
- 코스 주변 핀 6개: `피팅`, `측정`, `보관`, `물/간식`, `휴식`, `크루 모임`.
- 핀은 원형 오렌지, 라벨은 크림 패널.
- 하단 근거 패널 2개: `더현대 러닝 클럽`, `CU 한강 러닝 스테이션`.

주의:
- `더현대 535㎡/162평`, `CU 18개 점포 확대 계획`은 작은 근거 칩으로만 사용한다.
- 동네 모든 매장이 러닝 거점이 된다는 확대 해석 금지.

### 카드 5. Product

역할:
- 러닝제품 트렌드를 신발 하나가 아니라 사용 순간별 제품군으로 정리한다.

레이아웃 타입:
- Before-During-After 3단 제품 매트릭스.

구성:
- 배경: 다크 + `running-korea-2026-products.png` 부분 크롭.
- 중앙: 3열 매트릭스.
- 열 제목:
  - `Before`
  - `During`
  - `After`
- Before 항목: `러닝화 피팅`, `의류`, `양말`, `조끼`.
- During 항목: `웨어러블`, `기록 앱`, `수분 보충`, `에너지 간식`.
- After 항목: `리커버리 슈즈`, `회복용품`, `발 건강`, `보급/영양`.
- 각 열 상단에 작은 시간 라인: `뛰기 전` / `뛰는 중` / `뛰고 난 뒤`.

주의:
- 특정 제품의 의학적 효능, 부상 예방, 매출 전망 단정 금지.
- 브랜드별 거래액/검색량 수치는 출처 확인 전 그래프화하지 않는다.

### 카드 6. Checklist

역할:
- 작은 브랜드와 로컬 사업자가 저장해둘 수 있는 실행 관점으로 마무리한다.

레이아웃 타입:
- 2열 체크리스트.

구성:
- 상단: 짧은 저장형 제목.
- 중앙: 좌우 2열 패널.
- 좌측 제목: `기회`
- 우측 제목: `주의`
- 기회 체크 4개:
  - `크루가 모이는 장소인가`
  - `뛰기 전후 들를 이유가 있는가`
  - `기록/인증과 연결되는가`
  - `반복 구매가 가능한가`
- 주의 체크 3개:
  - `장비 과열을 부추기지 않는가`
  - `부상/안전을 과장하지 않는가`
  - `확인 안 된 숫자를 쓰지 않는가`
- 하단에 작은 `SAVE THIS` 캡슐을 둘 수 있으나 CTA처럼 크게 만들지 않는다.

주의:
- 사업 성과 보장, 창업 추천, `러닝만 붙이면 팔린다` 식의 표현 금지.
- `접수 경쟁`, `장비 과열`, `부상/안전 리스크`를 작은 주의 문구로 남긴다.

### 카드 7. Logo Ending

역할:
- 본문 종료 후 EARLYSHINE 브랜드 엔딩.

레이아웃 타입:
- 중앙 로고 엔딩.

구성:
- 배경: `#0F0E0C`.
- 우상단: 실제 로고 이미지 `assets/logo/main_a_transparent.png`, 본문 카드보다 살짝 작게 100~120px.
- 중앙: 같은 로고 이미지를 크게 배치, 권장 폭 360~460px.
- 중앙 로고 아래에는 불필요한 본문을 넣지 않는다.
- 하단 푸터에는 얇은 탄 분할선과 `07 / 07`만 최소 표기한다.

주의:
- 텍스트 `EARLYSHINE`로 대체하지 않는다.
- 추가 슬로건, 긴 문장, 장식 일러스트를 넣지 않는다.

## Writing에 넘길 공간 제약

- 제목은 카드당 18~28자 안팎, 최대 4줄.
- 보조 문장은 1~2줄, 42자 안팎.
- 도표 칩은 한 줄 8~15자.
- 카드 2와 4의 숫자는 근거 칩으로 작게만 사용한다.
- `1000만`, `1조`, `494건`, `270%` 등 확인 필요 숫자는 본문/도표 핵심 문구에 사용하지 않는다.

## Packaging 구현 제약

- 모든 카드는 1080x1350 독립 섹션으로 구현한다.
- 실제 로고 경로: `assets/logo/main_a_transparent.png`.
- 로고는 CSS 텍스트가 아니라 `<img>`로 삽입한다.
- PNG/PDF 출력 총 장수는 7장이다.
- 파일명은 packaging 단계 규칙에 맞춰 1~7장 모두 출력한다.
- 사진 크롭은 `object-fit: cover`를 쓰되 얼굴/제품 주요부가 잘리지 않게 `object-position`을 카드별로 조정한다.
- 도표는 가능하면 HTML/CSS 또는 SVG 라인워크로 구현하고, 텍스트는 이미지로 굽지 않는다.
- 표/칩 최소 글자 크기는 18px 이상 유지한다.

## 금지 스타일

- 귀여운 툰형 러너 캐릭터
- 손그림 코스 지도
- 파스텔 운동 기록장/다이어리 템플릿
- 보라색/파란색 그라데이션
- 과한 오렌지 CTA 배경
- 실제 앱 화면, 실제 브랜드 로고 무단 복제
- 긴 설명 문단
- 출처 불명 숫자를 메인 그래프로 쓰는 방식

## 최종 QA 체크리스트

- 7장 모두 우상단에 실제 EARLYSHINE 로고 이미지가 들어갔는가?
- 7장 마지막 카드가 중앙 로고 엔딩으로 보이는가?
- 카드 1만 봐도 "러닝은 동선 비즈니스"라는 프레임이 보이는가?
- 카드 3의 흐름도가 앱/인증 구조로 읽히는가?
- 카드 4의 미니맵이 러너 동선과 오프라인 거점을 설명하는가?
- 카드 5의 Before-During-After 제품 매트릭스가 한눈에 구분되는가?
- 카드 6이 저장용 체크리스트로 보이는가?
- 오렌지가 작은 강조에만 쓰였는가?
- 사진이 3장 이내이며 다크/웜 톤으로 통일됐는가?
