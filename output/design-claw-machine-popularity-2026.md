# Design Brief: 뽑기방이 인기있는 이유?

결론:
1. 전체 톤은 "유리 진열장 안의 미니 리포트"로 잡는다.
2. 사진 의존은 낮추고, 코드 기반 캡슐·유리·집게 라인워크와 데이터 패널로 7장을 통일한다.
3. 본문 6장은 우상단 실제 EARLYSHINE 로고, 7장은 중앙 로고 엔딩 카드로 구성한다.

근거:
- 주제가 장난감·랜덤 보상으로 흐르기 쉬워서, 귀여운 완구 무드보다 소비 행동 리포트처럼 보여야 신뢰도가 산다.
- Outline의 핵심은 "왜 인기인가"보다 "작은 돈, 즉시 결과, 소장 보상, 운영 문턱, 수집 시장, 리스크"라는 구조다.
- Warm/Tan 시스템은 숫자, 표, 체크리스트를 크게 보여줄 때 가장 강하므로 사진보다 선·면·도표 중심이 적합하다.

다음 단계:
- `writing`은 이 브리프의 카드별 슬롯에 맞춰 최종 문구를 짧게 확정하고, `packaging`은 HTML/CSS 구현과 PNG/PDF 출력만 수행한다.

## 전체 톤앤매너 한 줄

블랙 유리 진열장 위에 탄 컬러 라벨과 얇은 데이터 그래픽을 얹은, 컴팩트한 마케팅 스튜디오형 인스타 캐러셀.

## 캔버스

- 크기: 1080 x 1350.
- 장수: 본문 6장 + 로고 엔딩 1장, 총 7장.
- 안전 여백: 상하좌우 58~64px.
- 기본 배경: 단색 `#0F0E0C`.
- 내부 구성: 큰 제목 1개 + 핵심 시각 장치 1개 + 짧은 해석/주의 영역 1개를 기본으로 한다.
- 페이지 표기: 본문 카드는 `01/07`~`06/07`, 엔딩 카드는 필요 시 `07/07`를 아주 작게 둔다.

## 컬러 팔레트

- Base: `#0F0E0C` 전체 배경.
- Panel: `#16140F` 정보 패널, 차트 배경.
- Ink: `#F3EDDE` 제목과 본문.
- Muted: `#9F927A` 보조 설명, 출처, 축 라벨.
- Tan: `#D4B896` 핵심 숫자, 주요 라인, 강조 칩.
- Tan Deep: `#B79A6E` 보더, 보조 그래픽, 얇은 구분선.
- Line: `#514533` 카드 내부 격자와 도표 보조선.
- Orange Accent: `#D96035` 경고·민원·리스크 포인트에만 5% 이하로 제한.
- Glass Highlight: `rgba(243, 237, 222, 0.12)` 유리 반사선, 캡슐 하이라이트.

사용 비율은 Base 65%, Panel 20%, Ink/Tan 12%, Orange 3% 이하로 둔다.

## 폰트 규칙

- 기본 서체: Pretendard Variable 또는 Pretendard OTF.
- 메인 제목: 78~100px, 800~900, line-height 1.02~1.08.
- 카드 제목: 64~78px, 800~900, line-height 1.08~1.14.
- 핵심 숫자: 84~128px, 900, 숫자와 단위의 크기 차이를 둔다.
- 보조 문장: 26~34px, 600~700, line-height 1.35~1.5.
- 칩/라벨: 18~22px, 700~800.
- 출처/주의: 14~17px, 500~600.
- 전체 letter-spacing은 0으로 두고, 위계는 크기·굵기·색상으로 만든다.
- 큰 숫자는 단독으로 띄우지 말고, 숫자 바로 옆 또는 아래에 기준·단위·해석 문장을 붙인다.

## 카드별 레이아웃 타입

### 1. Hook

- 타입: 대형 제목 + 하단 3요소 루프.
- 구조: 상단 55%에 큰 질문형 제목 영역, 하단 35%에 `동전 -> 결과 확인 -> 손에 잡히는 보상` 흐름 그래픽.
- 시각 장치: 유리 뽑기 기계 창, 얇은 집게 라인, 캡슐 3개를 코드 기반 라인워크로 배치.
- 주의: 카지노·슬롯머신처럼 보이는 코인 폭발 연출 금지. "작은 돈의 즉시성"에 초점을 둔다.

### 2. Problem

- 타입: 오해 해체형 3분할 패널.
- 구조: 왼쪽에는 짧은 문제 제기 제목, 오른쪽에는 `가격`, `결과 확인`, `소장품` 3개 세로 패널.
- 시각 장치: 가격표 아이콘, 즉시 확인을 뜻하는 작은 윈도우, 물건 박스/캡슐 라인워크.
- 강조: "운" 하나가 아니라 경험 설계 요소가 쌓인다는 구조를 보여준다.

### 3. Domestic

- 타입: KPI + 문턱 낮추는 요인 스택.
- 구조: 왼쪽 45%는 `6,144곳` 대형 숫자 영역, 오른쪽 55%는 무인 운영·카드 결제·가성비 놀이문화 3단 스택.
- 시각 장치: 작은 매장 아이콘이 반복되는 밀도 그래픽, 카드 단말기 선 아이콘, 무인 입구 라인.
- 수치 처리: `6,144곳`은 Tan으로 크게 두되, 바로 아래에 기준·출처 영역을 붙인다.
- 주의: 수치만 떠 보이지 않도록 "문턱이 낮아졌다"는 해석 슬롯을 숫자 근처에 둔다.

### 4. Global

- 타입: 해외 맥락 2열 비교 + 중앙 브리지.
- 구조: 왼쪽은 일본 캡슐토이 맥락, 오른쪽은 Pop Mart/블라인드박스 맥락, 중앙에는 `취향 수집` 브리지 라벨.
- 시각 장치: 캡슐 그리드, 진열 선반, 블라인드 박스 실루엣을 모두 추상 라인워크로 처리.
- 브랜드 처리: Pop Mart는 텍스트 언급과 출처 표기만 허용. 실제 캐릭터, 로고, 제품 외형 모사는 금지.
- 강조: 해외 사례는 "유행 복제"가 아니라 "취향을 모으는 시장"이라는 맥락으로 보이게 한다.

### 5. Guardrail

- 타입: 리스크 대시보드.
- 구조: 상단에는 짧은 경고 제목, 중단에는 `공정성`, `경품 기준`, `민원 리스크` 3개 게이트, 하단에는 작은 체크 바.
- 시각 장치: 잠금 아이콘, 기준선, 알림 점, 신고/문의 라인을 최소한으로 사용.
- 컬러: Orange Accent는 경고 점과 얇은 라벨에만 사용한다.
- 주의: 겁주는 카드가 아니라 "운영 전에 확인할 조건"으로 읽히게 한다.

### 6. Takeaway

- 타입: Before/After + 체크리스트.
- 구조: 왼쪽은 따라 하면 안 되는 요소, 오른쪽은 가져와야 할 설계 요소. 하단에는 4개 체크리스트.
- 시각 장치: 왼쪽은 흐린 랜덤 캡슐 더미, 오른쪽은 설계된 보상 루프 다이어그램.
- 체크리스트 후보: 진입 가격, 보상 매력, 규칙 투명성, 재방문 이유.
- 강조: 마지막 본문 카드는 "랜덤"보다 "보상 설계"라는 실무적 결론이 남아야 한다.

### 7. Logo Ending

- 타입: 중앙 로고 엔딩.
- 구조: Base 배경 위 중앙에 `assets/logo/main_a_transparent.png`를 크게 배치.
- 시각 장치: 로고 뒤에 아주 얇은 Tan 라인 프레임 또는 유리 반사선 1~2개만 허용.
- 주의: 본문 문장, 장식 아이콘, 과한 CTA를 넣지 않는다.

## 카드별 시각 장치 요약

- 1장: 유리창 + 집게 + 캡슐 루프.
- 2장: 3요소 분해 패널.
- 3장: 대형 KPI 숫자 + 매장 밀도 아이콘 + 문턱 스택.
- 4장: 캡슐 그리드와 블라인드박스 진열 2열 비교.
- 5장: 공정성·경품 기준·민원 리스크 게이트.
- 6장: Before/After 보상 설계 다이어그램 + 체크리스트.
- 7장: 중앙 EARLYSHINE 로고.

## 반복 컴포넌트 규칙

- Header lockup: 좌상단 `카드 번호 + 섹션 라벨`, 우상단 실제 EARLYSHINE 투명 로고.
- Logo: 본문 카드 우상단에는 `assets/logo/main_a_transparent.png` 사용. 텍스트 대체 금지.
- Footer: 좌측은 출처/주의, 우측은 페이지 번호. 푸터 라인은 Tan Deep 35% 이하.
- Panel: `#16140F` 배경, 1px Tan Deep 35% 보더, radius 8px 이하.
- Chip: 라운드는 과하지 않게 999px 또는 6px 중 하나로 통일. 텍스트는 수직 중앙 정렬.
- Linework: 1.5~2px 선, Tan 또는 Muted, 반투명 유리 하이라이트는 매우 얇게.
- Data: 숫자, 기준, 단위, 해석 문장을 한 묶음으로 둔다.
- Icon: lucide 스타일에 가까운 단선 아이콘. 완구 캐릭터형 아이콘 금지.
- Rhythm: 각 카드의 제목 위치와 푸터 위치는 고정하고, 중앙 그래픽만 카드별로 변주한다.

## 사진/이미지 방향

- 기본 방침: 사진 의존 낮음. 코드 기반 SVG/CSS 라인워크가 우선이다.
- 제작 소재: 유리창 반사선, 캡슐 원형, 집게 암, 진열 선반, 카드 단말기, 매장 입구를 벡터/코드 그래픽으로 만든다.
- 사진을 꼭 쓸 경우: 실제 뽑기방 사진 1장 이하, 어둡게 듀오톤 보정, 배경 질감 정도로만 사용한다.
- 생성 이미지가 필요할 경우: "현실적인 무인 뽑기방 내부, 어두운 조명, 유리 기계, 사람이 없는 장면" 정도의 소재성 이미지로 제한한다.
- 사용 금지: 귀여운 인형 클로즈업, 특정 캐릭터 모사, 알록달록 캡슐 과다 노출, 손그림 삽화.

## 금지 스타일

- 귀여운 툰형, 손그림, 파스텔 노트 템플릿.
- 보라·파랑 그라데이션, 오브/블롭, 배경용 점무늬 장식.
- 장난감 광고처럼 보이는 무지개색 완구 팔레트.
- 카지노, 슬롯머신, 도박장 같은 과격한 비유.
- Pop Mart 또는 실제 캐릭터 IP의 로고·제품 외형 재현.
- 텍스트 `EARLYSHINE` 대체 로고.
- 긴 문단, 설명형 카드, 숫자만 크게 떠 있는 카드.
- 패널 안에 패널을 또 넣는 중첩 카드 구성.

## 최종 QA 체크리스트

- 1080 x 1350 기준에서 7장 모두 안전 여백을 지키는가?
- 본문 6장 우상단에 실제 EARLYSHINE 투명 로고가 들어갔는가?
- 7장 엔딩 카드가 중앙 로고 중심으로 정리됐는가?
- Warm/Tan 팔레트가 유지되고 Orange가 리스크 포인트에만 쓰였는가?
- 각 카드에 큰 메시지 1개와 시각 장치 1개만 남는가?
- `6,144곳` 등 수치에 기준·출처·해석이 붙어 있는가?
- 국내 맥락과 해외 맥락이 한 카드 안에서 섞이지 않고 구분되는가?
- 랜덤 보상 리스크가 과장 없이 공정성·경품 기준·민원 관리로 정리되는가?
- 모바일 축소에서도 제목, 숫자, 칩 라벨이 읽히는가?
- 캡슐·집게 라인워크가 장식이 아니라 메시지를 설명하는가?
- PNG/PDF 출력 후 한글, 로고, 얇은 선, 푸터가 깨지지 않는가?
- 패키징 이후 `design-review`가 실제 PNG/PDF 기준으로 P0/P1/P2 검수를 수행하는가?
